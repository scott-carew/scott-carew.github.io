CSS Display: Block, Inline & Inline-block

22 April 2022

Boxes boxes everywhere! If you’re new to web development, one of the first things you’ll learn is that essentially everything comes in the form of a box! Your headings are a box, your paragraphs are a box, those Sign In and Sign up buttons at the top of a page? Yup! Those are boxes too. Heck, even circles are boxes! And around those boxes, you guessed it - more boxes!

So how do we manipulate and manage all of these boxes? Well, a commonly used property in CSS is the display property which basically tells an element how to appear on a page. Whilst there are many different values that the display property can take on - I'm just going to cover a few of the basics here:

  • display: block
  • display: inline
  • display: inline-block

display: block

Before we dive headfirst into the inline and inline-block values, it’s helpful to understand block.

In CSS, many elements, by default, are block elements. What this means, is that when we have multiple elements, they appear on screen as a series of boxes sitting one on top of the other. Unless otherwise manipulated in some way, each element will take up the full width of the screen.

To see this in action, let's take a look at some cheesey divs! (The div element, by default has the display value block).

Can't we just adjust the width to make each div smaller so they can fit next to each other? You might be thinking...Well, we can adjust the width and height of block elements, however they're still going to sit on top of each other because that's just the way blocks roll!

Common examples of block elements include:

  • Paragraphs
  • All headings
  • Ordered and unordered lists
  • Articles, sections, divs

display: inline

Just like display: block, other elements in CSS default to display: inline and span is one of these! Now, I'm going to hazard a guess that you might be thinking - this means the elements will display in a line, right? Correct! Take a look at the cheesy span element below. A border has been placed around both the div and span elements for your viewing pleasure!

Now, the thing about inline elements, is that they tend to act like soldiers, and fall inline with the element that surrounds them. Changing the height and width (see 'blue castello' below) of the inline element has no effect. Whilst you can change the padding, and it will behave itself horizontally, take a look at what happens to 'cheese parmesan' in the image below when we do this. Chaos! This is basically the div element saying "I hear you, but I'm not listening! The padding simply covers the text above and below the line. No respect.

Common examples of in-line elements include:

  • Anchor tags (or links) < a >
  • Tags used to modify text inline such as < strong >, < em >, < u > and < mark >
  • < span >

display: inline-block

But we're an inclusive bunch! So what if we want to have elements display inline, and have their width, height and padding elements all respected because diversity is key! Well, this is where display: inline-block comes in.

As you can see in the image below, by setting the display property to the inline-block value in the CSS, the changes we made in the previous example, are now respected. We can easily adjust the height, width, vertical and horizontal padding without fear of covering our other text.

When creating buttons, display: inline-block is commonly used - especially navigation bars - where you want the ability to style the elements, but also have them appear horizontally. It can also be used to style sections of text within a paragraph, without having a negative impact on the other elements that it falls inline with.

So that's it! Three values of the element display done and dusted! Hopefully you now have a better understanding of the difference between display: block, display: inline and display: inline-block.
